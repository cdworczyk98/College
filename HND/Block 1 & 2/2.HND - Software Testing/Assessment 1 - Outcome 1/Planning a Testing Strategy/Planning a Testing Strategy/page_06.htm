Defect Testing

This test is intended to find areas where the program does not conform to its specification. Tests are designed to reveal the presence of defects in the system. When defects have been found in a program, these must be pinpointed and removed. This is called debugging. In fact, they are quite different. Testing establishes the existence of defects. Debugging is concerned with locating and correcting these defects. The debugger must generate hypotheses about the observable behaviour of the program and then test these hypotheses in the hope of finding the fault which caused the problem. Testing the hypotheses may involve tracing the code manually. Defect testing and debugging are sometimes considered to be part of the same process.

It may require new test cases to localise the problem. Interactive debugging tools, which show the intermediate values of program variables and a trace of the statements executed, may be used to help the debugging process.

After a defect in the program has been discovered, it must be corrected and the system should then be re-tested. This form of testing is called regression testing, which is used to check that changes to the program have not introduced new faults into the system.

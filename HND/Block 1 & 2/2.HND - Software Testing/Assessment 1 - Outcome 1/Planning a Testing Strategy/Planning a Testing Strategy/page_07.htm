The Testing Process

Except for small programs, systems should not be tested as a single, monolithic unit. Large systems are built out of sub-systems, which are built out of modules, which are composed of procedures and functions. The testing process should therefore proceed in stages, whereby testing is carried out incrementally in conjunction with system implementation. The most widely used process consists of five stages:

  1. Unit Testing
  2. Module Testing
  3. Subsystem Testing
  4. System Testing
  5. Acceptance Testing.

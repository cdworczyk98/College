Training

Any new system will certainly require user training to be carried out. If we have included the user in the specification, fact-finding, design and implementation phases of the system development, much of the training will already have been done.

It is generally preferable to carry out the training at the user's workplace since the equipment, documentation, procedures and data are to be found there. However, under certain circumstances it may be decided to take the user away from the workplace and to do the training in isolation. This may be the case where new techniques have to be learned by the user and we do not want them to be distracted by their normal workplace.

In the best-case scenario, the training of users would be done prior to the new system being implemented, but this is not always possible for business reasons. However, to avoid disaster, it is better if at least initial training is done. If the users are experienced, the training process can be speeded up.

Even when training has been carried out thoroughly and the new system is working perfectly, there can still be problems. The users should therefore have access to a user manual, which will answer common questions. This should be backed up by some user support, either online or by telephone, but preferably real time, especially in data-critical systems.

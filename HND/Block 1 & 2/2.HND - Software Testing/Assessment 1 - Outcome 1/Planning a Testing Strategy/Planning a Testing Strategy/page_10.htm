Black Box Testing

Black box testing means treating the software as though it were put inside a black

box (ie we don't know any details of what's inside). We do, however, know what data must be put into the system, and the results that should be output for each data input. We are thus concerned only with the fact that the system works, and not with the details of how it works. Black box testing is usually used during the software development phases, and ensures that all sensible data gives the right output, and all nonsensical data is rejected, or an appropriate error message is output.

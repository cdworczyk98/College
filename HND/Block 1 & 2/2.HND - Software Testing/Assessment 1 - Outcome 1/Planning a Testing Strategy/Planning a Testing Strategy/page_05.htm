Usability Testing

The usability testing applies to the visible part of the application. For example, we need to know if it is easy to use. Does it confuse the user? Does it follow the usual conventions? After all, if the users don't like the application, it doesn't matter how well it meets their requirements. The developer might have their own ideas about what is good, but at the end of the day, they must supply something that the user will be happy with.

We may also wish to include the time it takes to accomplish a task, ie usability. We may then have to re-design the code if a particular function takes longer than is acceptable to execute. In reality, the users must be involved in the process of testing the usability of the application as soon as possible, preferably at the design stage. After all, what the design looks like and how it works are both part of the design. In some cases we can create dummy forms just to show users what the application will look like.

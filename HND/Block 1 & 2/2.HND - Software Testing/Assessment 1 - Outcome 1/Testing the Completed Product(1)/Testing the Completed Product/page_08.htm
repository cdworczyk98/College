Testing Approaches - Black Box Testing

This approach to testing regards the implemented design as a black box, specified inputs are deemed to produce specified outputs.

Examples

1. If a person logged in to their web based email, specified inputs are likely to be the user id and password and specified outputs are likely to be a welcome message, such as 'Welcome XXXXX', and access to the user's email facility.

2. If we were testing (evaluating) software installed on a PC or workstation, we might plan inputs such as software license code and owner details in order to specify outputs such as 'XXXX software installed correctly'.

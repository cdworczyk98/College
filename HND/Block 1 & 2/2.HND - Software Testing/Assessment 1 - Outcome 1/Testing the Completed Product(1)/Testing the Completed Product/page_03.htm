Types of Tests

Pathological Data

If the user is required to enter data in a particular format such as a National Insurance number which has the format XX 99 99 99 X (where X represents an alphabetic character and 9 a numeric character). We want to ensure that this rule is followed. We would test the system by putting in the wrong data type for example 22 AA BB CC 3. This is an example of pathological data and we would expect the system to reject it. In fact certain application packages allow us to create a "mask" which effectively forces the user to enter the correct format. Pathological data can also include non-alphanumeric characters.

Range Data

This covers several areas. Assume that a field requires the user to enter a value from 1 to 5. We obviously want to check that it deals with this range but what if the user enters 0 or 6 or 9? We would want these to be rejected. This is just checking that only the correct range of values is entered. (We can also apply pathological tests to this type of field).

Transcription Errors

Imagine we have to enter someone's date of birth which is the 12 of July 1985. There are several formats for this in use. In the UK we would enter it in the format DDMMYY - 120785. However in the US they use MMDDYY - 071285. This is still a valid date but in the UK we would read this as 7 December 1985.

Errors of this nature would only come to light if, for example, we were entering a date of 24 May 1987. This could be input by the user as 240587 or 052407. Then it would be trapped. It is usually a good idea to let the user know the correct format, for example:

"Enter Birth Date as DDMMYY"

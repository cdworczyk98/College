Valid and Invalid Test Data

As you can see from the previous section, we do not always plan to test with 'correct' data. To really test our solution we need to plan for occasions when the user does not, for example, input the correct data. Such test cases and data are called invalid, here meaning test cases and data for which the design has not been created to handle.

An example from a plan to install software on a PC might be the test case where invalid data was input in response to the prompt 'Continue Y or N?'. The invalid data specified might be 'User hits backspace', and the expected outcome might be 'System does not respond, software not installed'. The expected outcome will depend on the requirements specified earlier in the development process.

Ideally, a solution will be robust enough to not accept invalid data without becoming unstable.

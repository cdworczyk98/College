Planning Black Box Testing - 2

There are other ways to plan for black box testing. For detailed designs, the post conditions can be used to specify test cases. Equivalence partitioning can be used at any level of design.

Equivalence partitioning

Each input domain (an input domain refers to all the possible inputs) can be divided into partitions. Partitions are decided by reference to categories or criteria. Any data drawn from the category, or partition, is deemed to be equivalent to any other from the same partition, or category. Instead of using many sets of data which meet the same criteria, just one set can be specified. This can reduce the amount of planning for testing but give the same level of confidence in the evaluation of the product.

To increase confidence further, data from the boundaries of the partitions can be used in the test plans.

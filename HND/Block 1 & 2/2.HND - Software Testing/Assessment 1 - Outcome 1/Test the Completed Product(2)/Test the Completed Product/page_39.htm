Testing

You must test the completed program using a top-down testing strategy.

Download and complete the testing documents by entering the appropriate reference for each test in the actual results box. Use the Print Screen facility to record actual results and number these accordingly. Attach the actual results to the testing documentation.

Internal Documentation

Add comments to your program, including a short description of what each procedure does.

At the very top of the program type the following:

 
{----------------------------------------------------------------------------------	}
{ Program Identification Section	}
{----------------------------------------------------------------------------------	}
{ Program FileName:	}
{ Program's name on disk, eg: TechSystem.dpr	}
{ Version No.: Numeric value that changes with each version	}
{ DateWritten: Date of the last change to the code, i.e today	}
{Author:  Name of the programmer, ie: you.	}
{ Language: The programming language used, ie: Pascal	}
{----------------------------------------------------------------------------------	}
{ Description: A brief description of the function of the program	}
{----------------------------------------------------------------------------------	}

For each procedure type the following:

 
{ Module Ref No: Reference number 	}
{ Module Name: The name of the procedure	}
{ Description: A brief description of the procedure	}

Sections of the Testing Report

In the documentation of the project given as assessment there are several sections, which should be included. These are:

  1. A Test Strategy - This should give an account of your testing as the program developed. It should also lay out clearly your systematic choice of test data and the reasons for doing so. It would not normally actually give the data although some examples can be given where it helps to explain the idea. It normally explain the ideas in the order in which they were carried out.
  1. Test Data with expected results. This time the actual data entered should be given. The data should correspond with the strategy already explained. You should always work out the expected answers in advance.
  1. The actual output of the test runs should be included to show that the expected results have been obtained.
  1. A diary of errors and testing should be kept. This should be entered in at the time that they occur. It should be a record of the development of the program in its 'error free' state.

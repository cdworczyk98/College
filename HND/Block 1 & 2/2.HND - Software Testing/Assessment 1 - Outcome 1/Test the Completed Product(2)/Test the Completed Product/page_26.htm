Ex 2: Run-Time Errors

This exercise involves the use of Free Pascal's Debugger to try and discover any part of a computer program that may result in errors occurring when the program is running, ie: Run-time errors. It uses the program you typed in for Exercise 1, ie: the successfully compiled RatioFinderSystem program

This program has a bug in it which causes the program to work incorrectly on the entry of a 0 to finish. The problem is a 'Division by zero' error which will cause the running program to crash.

You can use the Watch window part of the debugger to take at look at the variables declared in the program and the values they take as the program progresses.

  1. Click on Debug on the main menu and choose the Add Watch command.
  1. When the Watch box comes up, type Number1 and press [Enter]. You'll see that Number1 has now appeared in a Watch window, along with its current value, eg: <Unknown Value>.
  1. Now use the Add Watch command to add Number2 and Ratio to the Watch window.
  1. Next, use Add Watch to add the expression Number1 / Number2 to the Watch window.
  1. Choose Run and Trace Into, or press [F7], to step through your program.

This time when you enter the two numbers, enter 0 for the second number. When you press [Enter] and go back to the program, look at the expression Number1 / Number2, it has the phrase 'Invalid floating-point operation'. That's because dividing by zero is undefined.

Having this expression in your Watch window doesn't cause the program to stop with an error. Instead the error is reported to you and the debugger does not perform the division in the Watch window.

Now press [F7] again, assigning Number1 / Number2 to Ratio. At this point, your program does halt, and the error message appears at the top of the edit window again.

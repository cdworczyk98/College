Ex 2: Solution

The problem with the program is that if you enter a value of zero for Number2, the program halts with a run-time error. How can it be fixed?

One way is to test if Number2 has a value of zero, and if it does don't allow the ratio calculation to take place, ie: don't divide Number2 into Number1. The relevant section of code is as follows:

 
Write ('please enter the second number ('Enter 0 to finish): ') ;
Readln (Number2) ;

If (Number2 = 0)
Then
Begin
Writeln ;
Writeln ('The ratio is undefined.');
End
Else
Begin
Ratio := Number1 / Number2 ;
End ;

Amend your program and run it again using the debugger. Note how the values in the Watch window change as you step through the program. When you are ready to stop, enter 0 for Number2. The program will pause after printing the message 'The ratio is undefined and pressing the [Enter] key will exit the program as required.
